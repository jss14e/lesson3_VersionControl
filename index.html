Highlight #1 Version Control

Version control is very important in today's world. Most often, the files we create go through various revisions. After everything is said and done you are left with several saved files with very similar names. This can be extremely confusing especially when working in groups. If someone needs to look at a file you may not know which file they need. In addition, if someone else were to make edits to a file, what would the naming convention be and how would they send it back? To tackle these issues, we have implemented Version Control Systems (VCS). Under this system, all files are stored within one repository. People are able to access the files and edit them easily. Multiple people can edit the same file without overwriting each other which is a huge benefit. If there are the multiple revisions of a file they can also merger together to combine everyone's efforts. This makes the process of collaborating much easier.

Highlight #2 Git

Git is one of the most popular version control systems we have today. Git is locally enabled, meaning you can manage your different versions on a local machine. Git, however, also offers for version control to be done remotely. This allows for multiple people to work on the same project at the same time. When using Git, one would push their file to the repository. After this anyone else can pull the file onto their local machine. Once a file is pulled down, it can be edited to the person’s liking. If the updated file were to be pushed to the repository again everyone would be able to see the branch of new revisions. People can then look at it the revisions and continue off of it or ignore it and work on a different version.

Highlight #3 Git Commands

As we discussed earlier version control is very important when sharing files. Git is a very popular tool when it comes to version control. One of the first commands one would use is: $ git init nameofproject . This line of code creates the directory of the name nameofproject. Git add . prepares the files and makes them ready to be committed. It is within this line of text that the file changes get added in the working directory. Git commit would then be used to record the file at the moment of typing in the command. This historical snapshot is then pointed to in the repository.

Highlight #4 Local Repository

A repository is a place where people can save and edit their files. It’s also a place where people can track and view the status of a file at various points in time. Files usually go from the directory on the computer to the repository. In a local repository, the repository is on the local machine. Local repositories typically are for individual users because it can be difficult to collaborate through the use of one machine. You can create a local repository by creating a brand new repository on the local machine or you can clone and existing repository and put it on your machine.

Highlight #5 Remote Repository

While a local repository is typically for one user and put on a local machine, a remote repository is typically for multiple people and it is placed on a remote server. Because it is on the remote server it allows for multiple people to access it at the same time. With this repository multiple people can work on the same file without overwriting each other. Person A and person B could edit the same file and push it to the directory. Instead of the file getting overwritten by the second person, both are saved as separate branches. These branches can also be viewed and edited like the original file. The group can decide what revisions stay and which should no longer get worked on.
